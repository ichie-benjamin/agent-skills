# Project Memory

Decisions and lessons that a future agent could NOT infer from code or git log.
Keep entries terse. Each entry must include a "why" — if you can't articulate
why, the entry isn't worth saving.

Format:

```
## YYYY-MM-DD — Terse title
1–4 lines. The decision, the reason, the trade-off accepted.
```

Compaction triggers at 150 lines OR 40 entries. Run `/dev-guideline compact`.
